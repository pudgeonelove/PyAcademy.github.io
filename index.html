Урок 1: Базовые настройки

Конфигурация сервера (Переменные)

Любая система начинается с настроек. Переменные хранят параметры, которые будут использоваться во всем скрипте. Зададим базовые параметры.

init_node.py 
node_name = "Gateway-Alpha"
max_connections = 5000
firewall_active = True

print("Инициализация узла:", node_name)
print("Лимит соединений:", max_connections)
> Инициализация узла: Gateway-Alpha
> Лимит соединений: 5000

Вычисления нагрузки

Рассчитаем пропускную способность канала. Скрипт должен заранее знать, хватит ли нам ширины канала для передачи бэкапов.

bandwidth.py 
file_size_gb = 150
speed_mbps = 800

total_megabits = file_size_gb * 8192
transfer_time_sec = total_megabits / speed_mbps

print("Время передачи (сек):", transfer_time_sec)
> Время передачи (сек): 1536.0

Маршрутизация (If/Else)

Управление доступом. Если IP-адрес входящего пакета не входит в белый список, мы обязаны сбросить соединение.

firewall.py 
incoming_ip = "192.168.1.105"
allowed_subnet = "192.168.1."

if incoming_ip.startswith(allowed_subnet):
    print("Соединение установлено.")
else:
    print("Пакет отброшен (DROP).")
> Соединение установлено.

Очереди задач (Lists)

Списки идеальны для хранения логов или очередей процессов. Мы создадим очередь демонов, которые должны запуститься при старте сервера.

services.py 
services = ["sshd", "nginx", "postgresql", "docker"]

print("Служб в очереди:", len(services))
print("Приоритетная служба:", services[0])
> Служб в очереди: 4
> Приоритетная служба: sshd

Автоматизация (For)

Вместо того чтобы вручную проверять каждый порт, мы поручим это циклу. Он пройдет по всему массиву и выполнит сканирование.

scanner.py 
ports = [22, 80, 443, 3306]

for p in ports:
    print("Сканирование порта", p, "... ЗАКРЫТ")
    
print("Аудит завершен.")
> Сканирование порта 22 ... ЗАКРЫТ
> Сканирование порта 80 ... ЗАКРЫТ
> Сканирование порта 443 ... ЗАКРЫТ
> Сканирование порта 3306 ... ЗАКРЫТ
> Аудит завершен.

Модули кода (Функции)

Когда код повторяется, его оборачивают в функцию. Это как создание собственной консольной команды.

ping.py 
def ping_server(ip_address):
    print("Пинг узла:", ip_address)
    print("Ответ: 12ms")

ping_server("10.0.0.1")
ping_server("8.8.8.8")
> Пинг узла: 10.0.0.1
> Ответ: 12ms
> Пинг узла: 8.8.8.8
> Ответ: 12ms

Базы данных (Словари)

Словари хранят данные в формате «Ключ: Значение». Это идеальная структура для хранения профилей или конфигурации баз данных.

db_config.py 
db = {
    "host": "localhost",
    "user": "admin_root",
    "port": 5432
}

print("Подключение к хосту:", db["host"])
print("Порт:", db["port"])
> Подключение к хосту: localhost
> Порт: 5432

Отказоустойчивость (Try/Except)

Блок try/except позволяет перехватить сбой (например, отсутствие файла) и выполнить резервный код, чтобы система не "упала".

backup.py 
try:
    file = open("backup.zip", "r")
    print("Файл открыт")
except FileNotFoundError:
    print("ОШИБКА: Файл не найден!")
    print("Запуск резервного копирования...")
> ОШИБКА: Файл не найден!
> Запуск резервного копирования...
